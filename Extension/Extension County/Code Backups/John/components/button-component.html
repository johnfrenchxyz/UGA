Button Component

The purpose of this component is meant to allow users to add buttons to their page easily with various cusomization options. These buttons are included in Foundation by default, so the AEM dialog options will primarily be adding/removing the various CSS class options illustrated below.

Other Info/Rules:

  • Foundation Button Documentation
  • For the sake of consistency, I usually use anchor tags with the "button" class for normal links and then i use the button form elements.
  • It might be worth our time to call the button color options by their actual color in the dialog instead of their class name. So, "alert" button would be called "blue" button in the dialog.

Accessibility"

    Use the role="button" to make buttons more accessible. You can use an aria-label="submit form" attribute to give a screen reader the literal purpose of the button if there is no text in it. If there is no href in the link then simply add the tabindex="0" to the div or span to make it focusable.
